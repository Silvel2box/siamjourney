เว็บท่องเที่ยวไทย "SiamDiscovery/SiamExplore" จัดเนื้อหาแบบ ภาค → จังหวัด(77) → หมวด(สถานที่ท่องเที่ยว/ร้านอาหาร/คาเฟ่/สินค้า OTOP). เริ่มภาษาไทยก่อน วางโครงรองรับ i18n (`/en/` + hreflang) ทีหลัง — ไม่พึ่ง Google Translate widget เพราะเสีย SEO.

**ตัดสินใจแล้ว (2026-07-14):**
- Stack: **Next.js (App Router, TypeScript) + Tailwind** — เลือกแทน Astro เพราะต้องการ portal ร้านค้า/คอมมิชชั่น (เฟส 2) อยู่ในแอปเดียวกัน และรับได้กับการรัน Node runtime บน Plesk (Passenger). ดึงข้อดี SEO ของ Astro มาด้วย: default = React Server Components (near-zero client JS), `generateStaticParams` gen หน้า 77 จังหวัด + place เป็น static, Metadata API + hreflang, app/sitemap.ts + robots.ts, JSON-LD schema.org. กฎโปรเจกต์: default Server Component เสมอ, ใส่ `"use client"` เฉพาะที่จำเป็น (search, mobile menu). markdown loader เอนไปทาง **Velite** (type-safe, เบา) — ยังไม่ล็อกสุดท้าย.
- Deploy Plesk: เฟส 0-1 เนื้อหา static แต่รันเป็น Next app (`next start`) เตรียมรับ API เฟส 2. next/image บน Plesk ต้องตั้ง loader หรือ unoptimized.
- Monetization MVP: **Affiliate-first** (Agoda/Booking/Shopee/Lazada) ผ่าน `<AffiliateButton>`, เว้น `<AdSlot>` ไว้รอ AdSense, ใส่ field `sponsored: 0/1/2` ใน frontmatter เตรียมระบบขายพื้นที่/คอมมิชชั่นทีหลัง (ยังไม่ทำ backend ร้านสมัครในเฟสแรก).

**ที่มา:** เริ่มจากไฟล์เดียว `thailand_travel_template.html` (Tailwind CDN, landing page โทนทอง/ดำ, ฟอนต์ Kanit+Prompt) → จะกลายเป็น BaseLayout + components.
